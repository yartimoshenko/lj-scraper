# LiveJournal Archive Scraper

Скрипт для скачивания полного архива блогов LiveJournal — посты, комментарии, картинки — в локальные HTML-файлы с навигацией по годам, тегам и полнотекстовым поиском.

## Возможности

### Скачивание
- **Несколько авторов** — можно указать через запятую: `https://palaman.livejournal.com/, https://krylov.livejournal.com/`
- **Один пост** — можно указать URL конкретного поста: `https://palaman.livejournal.com/12345.html`
- **Выбор годов** — конкретные годы (`2015,2016`), диапазон (`2012-2018`) или все годы (оставить пустым)
- **Полная загрузка комментариев** — из JSON-объекта `Site.page`, без браузера и UI-взаимодействия
- **Сохранение картинок** — все изображения из постов и комментариев скачиваются локально
- **Внешние посты** — если автор ссылается на пост другого ЖЖ-пользователя, этот пост тоже скачивается (один уровень вглубь)
- **Кросс-ссылки между авторами** — ссылки между скачиваемыми авторами автоматически заменяются на локальные
- **Обход anti-bot защиты** — `curl_cffi` с Chrome TLS impersonation, единая async-сессия

### Навигация
- **Главная страница** (`01.MAIN.html`) — список всех авторов, общие годы, теги, полнотекстовый поиск
- **Страница автора** (`author/01.MAIN.html`) — годы, теги и поиск для конкретного автора
- **Страницы по годам** — хронологический список постов за год
- **Страницы по тегам** — все посты с данным тегом
- **Полнотекстовый поиск** — клиентский JavaScript-поиск по всем постам (минимум 5 символов)

### Инкрементальное обновление
- **Пропуск скачанных** — при повторном запуске уже скачанные посты пропускаются автоматически
- **Перекачка** — можно выбрать перекачку существующих постов (если автор обновил записи)
- **Кеш 404** — удалённые посты внешних авторов запоминаются в `ext_failed.txt` и не качаются повторно
- **Retry** — автоматические повторные попытки при ошибках, адаптивные паузы при rate-limiting

## Установка

```bash
pip install curl_cffi lxml
```

Требуется Python 3.12+.

## Запуск

### Основной режим
```bash
python lj_scraper_scrapling.py
```

Скрипт интерактивно спросит:
1. URL автора (или нескольких через запятую)
2. Папку для сохранения
3. Годы для скачивания

### Пересборка индексов (без скачивания)
```bash
python lj_scraper_scrapling.py --reindex C:\путь\к\архиву
```

Пересобирает только навигационные страницы и поисковые индексы — полезно после исправлений в коде генерации.

## Как это работает

1. **Сбор списка постов** — для каждого автора парсятся страницы по годам → месяцам → списки постов
2. **Скачивание постов** — async-загрузка с семафором (4 параллельных запроса), извлечение заголовка, тегов, даты, контента
3. **Загрузка комментариев** — из JavaScript-объекта `Site.page` на странице поста (не из HTML), thread-URL для полного дерева
4. **Скачивание картинок** — все `<img>` из постов и комментариев сохраняются локально
5. **Кросс-ссылки** — внутренние ссылки между авторами заменяются на локальные пути
6. **Внешние посты** — ссылки на посты сторонних ЖЖ-авторов скачиваются (один уровень)
7. **Генерация индексов** — навигационные страницы по годам/тегам, поисковый индекс

## Технические детали

- **HTTP**: `curl_cffi` с Chrome TLS impersonation — единая `AsyncSession` на весь прогон (как настоящий браузер)
- **Парсинг HTML**: `lxml` с автодетектом кодировки (UTF-8 / Windows-1251)
- **Комментарии**: JSON из `Site.page` → thread URL для полного дерева вложенных комментариев
- **Параллелизм**: `asyncio.Semaphore` — контролируемая конкурентность без блокировки
- **Лог**: весь вывод дублируется в `scraper_log_*.txt` в папке архива
- **Поддержка разных тем ЖЖ**: fallback-парсинг для альтернативных шаблонов (без `#alpha-inner`, без `viewsubjects`)

## Ограничения

- Скачиваются только публичные посты (без авторизации)
- Внешние посты скачиваются на один уровень вглубь (ссылки внутри внешних постов не раскрываются)
- При большом количестве постов (10 000+) поисковый индекс может быть большим (~100 МБ)
